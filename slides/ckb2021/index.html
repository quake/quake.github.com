CKB 2021 Hardfork

By @quakewang

矿池

  • 不涉及到经济模型改变, 容易协调 (微信群, telegram group)
  • 挖矿节点一般都会跟随版本升级
  • 升级可以通过切换一点点算力进行在线测试
  • 沟通小技巧: 性能, 分叉
  • 难点: 分账程序的升级

矿池分账程序

矿池分账每家都有各自的方案, 而且按接入 CKB 节点的时间点不同, 可用的SDK和服务也不同, 形成了五花八门的方案, 猜猜看下列哪一个方案没有被矿池使用

  • A. ckb-cli api server
  • B. sdk + mercury
  • C. sdk + ckb-indexer
  • D. pure ckb rpc

交易所

  • 单独的沟通渠道
  • 一般不会跟随版本升级
  • 沟通小技巧: 分叉, 安全漏洞
  • 难点: DDD (deadline driven development), CKB, SDK 版本老旧

交易所 CKB 版本老旧

猜测一下所有的交易所中在这次 hardfork 之前运行的 CKB 版本最老的是什么:

  • A. 0.40.0
  • B. 0.41.0
  • C. 0.42.0
  • D. 0.43.0
  • E. 0.100.0
